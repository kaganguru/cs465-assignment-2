cs 465 assignment 2 prompt

For this assignment, you are going to develop a 3D hierarchical model viewing and animating application using raw WebGL. You will use the matrix stack and the tree traversal approaches (recursive traversal of matrixes for hierarchical modeling) that are used to manipulate the primitives and transformations using the matrix stack.

Rendering

  • webgl glsl shaders
  • It should have a basic diffuse and specular reflection calculation. We don’t have normal maps and other pbr attributes. Just map the texture and do a fragment shade with very default diffuse and specular reflection.
  • Light is directional so it can be a uniform variable passed to the shader as a vector.
  • rotation values in the animaton should rotate the parts around their pivot points.
  • the application should calculate the final transformation matrix and send it to the shader.
  • the application also calculates the projection matrix for the final render obviously.

Hierarchical Modeling

I exported these models into the ‘assets’ folder and it outputted an obj file and an mtl file for each part. See the content of these files for double check. And these models have textures, which for all of the body parts, they are stored in the textures folder. Mtl files reference to these textures.

Robot hierarchy (scene graph):

  • Body is the root.
  • Depth 1 children of Body: Head, UpperLegFL, UpperLegFR, UpperLegBL, UpperLegBR.
  • Each UpperLeg* has one child: its matching LowerLeg* (depth 2).

So the tree is:

Body
 ├─ Head
 ├─ UpperLegFL ── LowerLegFL
 ├─ UpperLegFR ── LowerLegFR
 ├─ UpperLegBL ── LowerLegBL
 └─ UpperLegBR ── LowerLegBR

  • Implement the robot as a tree rooted at Body, draw with a recursive function using push/pop around each node so that each child (head, upper legs, lower legs) inherits the full transform chain from its parent instead of being placed independently in world space.
  • 1. Use a recursive draw function with a matrix stack. Core rule:
    • Each child must inherit all transforms from its parent.
    • Use a push/pop (or equivalent) around each node. Some pseudo codes for you:
      function drawBody(params):
    pushMatrix()
        applyBodyTransform(params.body)      // position + rotation of whole robot
        drawBodyMesh()

        drawHead(params.head)
        drawUpperLegFL(params.legFL)
        drawUpperLegFR(params.legFR)
        drawUpperLegBL(params.legBL)
        drawUpperLegBR(params.legBR)
    popMatrix()


      function drawUpperLegFL(legParams):
    pushMatrix()
        applyRotation(legParams.upperAngle)
        drawUpperLegFLMesh()
        drawLowerLegFL(legParams)
    popMatrix()

function drawLowerLegFL(legParams):
    pushMatrix()
        applyRotation(legParams.lowerAngle)
        drawLowerLegFLMesh()
    popMatrix()

    Repeat the same pattern for the other legs and the head.

  • 2. Rules the AI MUST follow
    • Never reset the transform per part.
      • Do not place head/legs directly in world space.
    • They must be drawn under the Body transform.
    1. Always:
      • pushMatrix()
      • apply this node’s rotation/translation
      • draw its mesh
      • call children
      • popMatrix()
    1. All body movement automatically affects everything:
      • When Body moves/rotates, Head and all legs move with it.
      • When an UpperLeg rotates, its LowerLeg follows.
    1. Per frame:
      • Rebuild transforms from scratch using the current joint angles (no cumulative updates)

Key Frame editor

  • in the keyframe editor there will be 10 rows corresponding the key frames of transformations of each body part : “Head, Body, Front Right Upper Leg, Front Right Upper and so on for other 3 legs…)
  • each keyframe holds a translation (vec3) and a rotation (vec3) value.
  • from keyframe to other keyframe these values will be linearly interpolated for animation.
  • if there are no keyframes, the values for the transformations should be set to default. these default values will be constants I will play with in the code to set up the default positioning/rotation of body parts.
  • keyframes will have percentage value so their position is determined by at what percent of the animation they are. so if the animation duration is 5 seconds and a keyframe is at the 50% position it’s gonna be at second 2.5. You can see the ruler at the top of the keyframe editor starting from 0 and ending at 100.
  • at the top of the keyframe editor add a number input for animation duration. default is 5 seconds and changes to this should appear immediately. so if the current frame is at 40% and I set this value from 5 to 10 it should reach to end at 6 seconds (to complete the remaining 60%)
  • current frame will increase (100/duration) percent in a second.
  • animation will loop, and interpolation will be done as the last frame is the first frame so there’s no cuts, while looping.
  • at the top of the keyframe editor we’ll have this 3 buttons
    • Play button resumes the animation (current frame can continue to increase)
    • Pause button pauses the animation (current frame should stop)
    • Reset button resets the current frame value to 0 percent, which is resetting.
  • we should be able to travel in the animation clicking or click dragging on the time ruler

Adding/Editing a key frame

  • you cannot add edit delete keyframes when the animation is playing. so, disable all the actions when the animation is in the playing state.
  • when, a click is clicked at a row at a certain frame position, it will add a new keyframe at the click position for that row/part. And it’s at clicked state, to start editing it right away
  • when the new keyframe is created at that position, initial transform values of the keyframe will be whatever’s supposed to be at that frame position. this way it will not break the continuity of the animation when a new keyframe is added.
    • so when there’s no keyframe we told that it will play the default position and wherever we click it will create a keyframe with this values.
    • and when there’s some keyframe and when we add the new one, initial values will be the interpolated value (whatever’s supposed to be at that frame position) So it’s like peeking the values of transform of animation at that position.
  • when a key frame is clicked state the value editor will show the inputs for the transformation values. these inputs should be a slider and should have a reset value that will reset them to to their initial value. you can edit these values here:
    • rotation x, y, z
    • translation x, y, z
  • you can also press delete key while a keyframe is in clicked state for editing, this will delete the keyframe.
  • when a keyframe is clicked to edit, or when a new keyframe is added at a new position current_frame teleports to the frame position of this keyframe. this the current render will be done with transform values corresponding to this exact frame. so if a keyframe is in clicked state we can safely tell the current frame is at the frame position of that keyframe.
  • when a keyframe is in clicked state, edit the tint value of the corresponding part to red to show which part you’re editing. this means that the object is in picked state. in picked state in addition to the value editor, there will appear a gizmo for transforming at the part’s origin point. (talked in the viewport editing section)

Viewport editing

  • when animation is not playing, user can pick by clicking on the body part. if already a keyframe is selected, you cannot re-click to the body part that keyframe is associated to. this is important because
  • this tutorial is provided for picking tutorial: https://webglfundamentals.org/webgl/lessons/webgl-picking.html
  • when a part is picked and transformed through the control gizmos (a gizmo including 3 arrows for translation and 3 rings for rotation ) it creates a new keyframe with a frame position value of current_frame
  • control gizmo helps editing the relative translation and rotation value by clicking on rings or arrows and dragging.
  • you can also edit this transform values in the value editor vice versa.
